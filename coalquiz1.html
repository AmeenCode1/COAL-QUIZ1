4 Functions of a Computer

  • Data Processing – Transforming data into meaningful info
  • Data Storage – Files stored for retrieval/update
  • Data Movement – I/O to/from peripherals; data comm
  • Control – Control unit manages all three above

Computer Structure (Top-Level)

  • CPU – Controls operations, processes data
  • Main Memory – Stores programs & data at runtime
  • I/O – Moves data to/from external environment
  • System Interconnect – System bus connecting above three

CPU Structure

  • Control Unit (CU) – Interprets instructions, sends signals
  • ALU – Performs arithmetic & logic operations
  • Registers – Internal CPU storage
  • Internal Bus – Connects CU, ALU, Registers

IAS Registers – ALU

  • MBR – Memory Buffer Register: holds 40-bit word to/from memory
  • AC – Accumulator: holds MSBs of ALU result
  • MQ – Multiplier Quotient: holds LSBs of result

IAS Registers – Control Unit

  • MAR – Memory Address Register: 12-bit memory address
  • IR – Instruction Register: 8-bit opcode
  • IBR – Instruction Buffer Register: holds right-hand instruction
  • PC – Program Counter: address of next instruction pair

Instruction Cycle

  • Fetch Cycle – PC → MAR → fetch word → MBR → IBR/IR; opcode to IR, address to MAR
  • Execute Cycle – Control circuit interprets opcode, sends signals to perform operation
  • IAS Instruction Types – Data transfer, Unconditional branch, Conditional branch, Arithmetic, Address modify

Computer Generations

  • 1st Gen – Vacuum tubes (ENIAC, UNIVAC)
  • 2nd Gen – Transistors (smaller, cheaper, less heat)
  • 3rd Gen – Integrated Circuits / ICs (1958)
  • 4th Gen – Microprocessors (1971–Intel 4004)
  • 5th Gen – AI / parallel computing

Moore's Law

  • Observation (1965) – Transistor count doubles every 12 months (later slowed to 18 months)
  • Consequence 1 – Cost per chip stays almost constant
  • Consequence 2 – Closer components → shorter paths → higher speed
  • Consequence 3 – Less power/heat dissipation
  • Consequence 4 – More reliable interconnections

Performance Balance

  • Problem – Memory speed lags behind processor speed → processor stalls
  • Memory Solutions – Wider DRAMs, cache buffer, more cache on chip, higher-speed buses
  • I/O Solutions – Cache/buffer schemes, high-speed interconnect buses, multiple processors

Processor Speed Techniques

  • Pipelining – Overlap instruction stages (fetch→decode→execute)
  • Branch Prediction – Predict next branch to preload instructions
  • Data Flow Analysis – Identify dependencies to avoid delays
  • Speculative Execution – Execute ahead speculatively, hold results
  • Superscalar – Multiple instructions execute per clock cycle

MIPS & CPI Formulas

  • T = IC × CPI × τ – Processor time (IC=instruction count, τ=cycle time)
  • CPI = Σ(CPIᵢ × Iᵢ/IC) – Average cycles per instruction
  • MIPS = f / (CPI × 10⁶) – Instructions per second in millions
  • Clock Rate – Alone doesn't indicate performance; compare within same architecture

Multicore Processors

  • Definition – Multiple processors on single chip with shared cache
  • Benefit – Doubling cores ≈ doubles performance (with software support)
  • Hyper-threading – Single core appears as 2 to OS
  • Turbo Boost – Cores automatically run faster than base frequency
  • Multi-threading – OS utilises multiple cores for application threads

32-bit vs 64-bit

  • 32-bit – Max 4GB RAM (2³² bytes), 32-bit registers/buses
  • 64-bit – Handles much larger RAM; needed for CAD, DB, servers
  • Compatibility – 32-bit software runs on 64-bit OS via emulator

Von Neumann / IAS Machine

  • Stored-program concept – Instructions and data in same memory
  • Memory – 4096 words × 40 bits each
  • Instruction format – Two 20-bit instructions per word; 8-bit opcode + 12-bit address
  • Harvard vs Von Neumann – Harvard uses separate buses for instructions & data → faster
